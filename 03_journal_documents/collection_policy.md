# Journal Document Collection Policy

This policy defines how journal-document evidence should be collected, preserved, and indexed for the architectonic review study.

## Purpose

The goal is to preserve enough evidence to support later coding, auditing, and reproducibility. Raw collection should preserve what was encountered at the source. Later ontology-driven extraction should be treated as a distinct downstream stage, not conflated with raw intake.

## Preservation Standard

For each collected journal document, preserve as many of the following as the source permits:

1. a source snapshot of the page or file,
2. a full-page screenshot when the rendered context matters,
3. a relevant excerpt screenshot when the decisive evidence is a visible subsection,
4. a raw text capture when visible text is copied from the source page,
5. an extracted text file produced later for ontology-driven coding,
6. the collection metadata in `documents_index.csv`.

When HTML or PDF preservation works cleanly, retain the source snapshot. When the page is dynamically rendered, omitted from print/PDF, or otherwise hard to preserve, capture the visible text as a raw text artifact and preserve a screenshot of the relevant rendered section. The later `extracted_text` layer should be reserved for text organized according to the study ontology during coding preparation.

## Controlled Vocabulary for `extraction_method`

Use one of the following values:

```text
html_text
pdf_text
browser_print_pdf
browser_copy_paste
browser_copy_paste_with_screenshot
screenshot_manual_transcription
ocr
manual_note_no_text_extracted
```

For a dynamically rendered or difficult page where copied visible text is paired with a screenshot, the preferred value is:

```text
browser_copy_paste_with_screenshot
```

## Minimal Workflow for Hard Pages

1. Open the live page in the browser.
2. Record the URL and access date.
3. Save a source snapshot if feasible.
4. Save a full-page screenshot when contextual rendering matters.
5. Save a relevant excerpt screenshot for the visible evidence actually used in coding.
6. Copy the relevant visible text into a plain-text raw text capture.
7. Record the extraction problem and any manual extraction notes.
8. Hash every preserved artifact and record the hashes in the document index.

## Folder Convention

Per-sampled-journal directories are generated by `03_journal_documents/generate_sample_artifact_folders.py`.
Starter index rows can be generated from the current sample with `03_journal_documents/prepopulate_documents_index.py`.
One-off or small-batch artifact intake can be done from `temp/journal_document_intake/record_journal_document_receipts.py`.

For each sampled journal, preserve files in this structure:

```text
03_journal_documents/
  raw/
    S0001_J0017_example_journal/
      source_snapshot/
      raw_text/
      screenshots/
        full_page/
        relevant_excerpt/
  extracted_text/
    S0001_J0017_example_journal/
```

## Rationale

This three-layer preservation standard strengthens the archive by retaining:

1. the source/page reference,
2. a visual record of rendered evidence,
3. a raw text capture that preserves visible source language,
4. a later extracted-text layer that can be organized around the study ontology.

That is substantially stronger than copy-paste alone when later methodological questions arise.

## Temp Intake Workflow

Use the temp workspace when you want to process a single document or a small group of artifacts without editing the canonical index by hand.

### Single-document CLI mode

Provide the journal metadata and whichever artifact files you already have in the temp workspace. Use `document_id` when you want to update a specific existing row. If you omit `document_id`, the script creates a new raw-artifact row and `document_type` can remain blank until extraction or coding clarifies how the source should be classified.

```powershell
python .\temp\journal_document_intake\record_journal_document_receipts.py \
  --document-id D0005 \
  --journal-id J0017 \
  --journal-name "American Philosophical Quarterly" \
  --url "https://example.org/editorial-policy" \
  --access-date 2026-05-19 \
  --extraction-method browser_copy_paste_with_screenshot \
  --full-page-screenshot .\incoming\apq_full.png \
  --relevant-excerpt-screenshot .\incoming\apq_excerpt.png \
  --raw-text .\incoming\apq_editorial_policy.txt
```

### Batch mode

Populate `temp/journal_document_intake/batch_template.csv` with one row per document record. Relative input paths are resolved from the CSV location.

```powershell
python .\temp\journal_document_intake\record_journal_document_receipts.py \
  --batch-csv .\temp\journal_document_intake\batch_template.csv
```

### Dry-run mode

Use `--dry-run` to validate matching, path resolution, and receipt naming without copying files or rewriting `documents_index.csv`.
